# ST-WBM 扩展 — 第四类：行为与工具命令

> 这一类命令控制条目的高级触发行为（递归、效果），  
> 以及打开管理界面和查看帮助的入口工具命令。

## 20. `/wb-set-recursion` — 批量设置递归控制

批量修改条目的三个递归相关字段，控制条目在递归扫描流程中的行为。  
递归扫描是 SillyTavern 世界书的高级功能：已触发的条目可以扫描其他条目并进一步激活它们。

### 用法

```
/wb-set-recursion name=<世界书> [pi=<true|false>] [po=<true|false>] [uids=<UID列表>]
```

### 参数详解

| 参数 | 字段名 | 含义 |
|------|--------|------|
| `pi=` | `Prevent Incoming` | `true` = 此条目**不被**其他条目的递归触发（阻止递归传入） |
| `po=` | `Prevent Outgoing` | `true` = 此条目**不会触发**其他条目的递归扫描（阻止递归传出） |

两个参数均为可选，未填写的字段保持原值不变。

### 常用场景与参数组合

| 场景 | 参数 | 说明 |
|------|------|------|
| 关闭所有递归限制（默认行为） | `pi=false po=false` | 条目完全参与递归链 |
| 触发后不扩散递归 | `pi=false po=true` | 此条目不会递归激活其他条目 |
| 只由关键字直接触发，不被递归传入 | `pi=true po=false` | 屏蔽来自其他条目的递归激活 |
| 完全隔离某条目不参与递归链 | `pi=true po=true` | 既不被递归触发，也不触发递归 |

### 示例

```
# 关闭所有条目的递归控制（恢复默认）
/wb-set-recursion name=MyWorldBook pi=false po=false

# 让某些背景条目不触发递归扩散
/wb-set-recursion name=MyWorldBook po=true uids=10,11,12

# 让某个顶层条目不被递归触发（只由关键字直接触发）
/wb-set-recursion name=MyWorldBook pi=true uids=20

# 完全隔离一批条目
/wb-set-recursion name=MyWorldBook pi=true po=true uids=30,31,32
```

### 字段值说明

- 值只接受 `true` 或 `false`（大小写敏感）
- 两个参数均可单独指定，互不影响

## 21. `/wb-set-effect` — 批量设置效果控制

批量修改条目的 Sticky（粘性）、Cooldown（冷却）、Delay（延迟）三个效果字段，  
控制条目在激活后的持续和等待行为。

### 用法

```
/wb-set-effect name=<世界书> [sticky=<n>] [cooldown=<n>] [delay=<n>] [uids=<UID列表>]
```

### 参数详解

| 参数 | 字段名 | 含义 | 单位 |
|------|--------|------|------|
| `sticky=` | `Sticky` | 粘性：条目触发后**保持插入**的轮数 | 整数，0 = 不粘 |
| `cooldown=` | `Cooldown` | 冷却：条目触发后**多少轮内不再触发** | 整数，0 = 无冷却 |
| `delay=` | `Delay` | 延迟：条目满足触发条件后**等待几轮**才真正插入 | 整数，0 = 立即 |

所有参数均可单独指定（未填的字段保持原值）。

### 常用场景与参数组合

| 场景 | 参数 | 效果 |
|------|------|------|
| 重置所有效果（默认行为） | `sticky=0 cooldown=0 delay=0` | 条目正常按关键字实时触发 |
| 重要信息触发后持续出现 | `sticky=3` | 触发后连续出现 3 轮，无论关键字是否继续出现 |
| 防止某条目频繁刷屏 | `cooldown=5` | 触发后 5 轮内不会再次触发 |
| 让信息延迟出现 | `delay=2` | 关键字出现后等 2 轮才插入 |
| 剧情触发器（触发一次后静默）| `sticky=1 cooldown=99` | 触发后出现 1 轮，之后很长时间不再触发 |

### 示例

```
# 重置所有条目的效果（清除所有 sticky/cooldown/delay）
/wb-set-effect name=MyWorldBook sticky=0 cooldown=0 delay=0

# 让 UID 1-3 的重要背景条目触发后持续 3 轮
/wb-set-effect name=MyWorldBook sticky=3 uids=1,2,3

# 让某些条目有冷却时间，避免反复出现
/wb-set-effect name=MyWorldBook cooldown=5 uids=10,11

# 让触发器类条目延迟生效
/wb-set-effect name=MyWorldBook delay=1 uids=20,21

# 复合设置：粘性 + 冷却
/wb-set-effect name=MyWorldBook sticky=2 cooldown=3 uids=30
```

## 22. `/wb-ui` — 打开双面板管理界面

在新标签页中打开 WorldBook Manager 的双面板可视化管理界面。  
这是进行复杂编辑操作（查看条目全文、批量勾选、拖拽排序）的推荐方式。

### 用法

```
/wb-ui
```

### 无参数

此命令不接受任何参数。

### 界面功能概览

```
┌──────────────────────────┬──────────────────────────┐
│ 左面板                   │ 右面板                   │
│ [选择世界书 ▼]  📤       │ [选择世界书 ▼]  📤       │
│ [搜索...]                │ [搜索...]                │
│ ☐ UID  标题             │ ☐ UID  标题             │
│ ☐ UID  标题（点击展开）  │ ☐ UID  标题（点击展开）  │
│                          │                          │
│ [批量操作▼] [← 复制]    │ [批量操作▼] [复制 →]    │
│ [＋新建] [＋＋批量新建]  │ [＋新建] [＋＋批量新建]  │
│ [💾 保存]               │ [💾 保存]               │
└──────────────────────────┴──────────────────────────┘
```

### 核心操作

| 功能 | 操作方式 |
|------|---------|
| 查看 / 编辑条目内容 | 点击列表中的条目展开内嵌编辑器 |
| 多选条目 | 勾选左侧复选框 |
| 批量操作已选条目 | 点击「批量操作」下拉菜单 |
| 跨面板复制条目 | 多选后点击「← 复制」或「复制 →」 |
| 保存修改 | 点击「💾 保存」（修改暂存内存，需手动保存才写入文件） |
| 下载当前世界书 | 点击世界书名称旁的 📤 按钮 |

### 保存机制

- 所有通过 UI 做的修改（编辑内容、批量操作）都**暂存在内存中**
- 关闭或切换世界书前，如有未保存的修改会弹出警告
- 点击「💾 保存」才会将修改写入服务器文件

### 与命令的关系

| 需求 | 推荐方式 |
|------|---------|
| 查看 / 编辑条目正文 | 使用 `/wb-ui` |
| 批量修改单一字段 | 使用命令（更快） |
| 对比两个世界书的条目 | 使用 `/wb-ui`（左右面板各选一个） |
| 拖拽调整条目顺序 | 使用 `/wb-ui` |

## 23. `/wb-help` — 显示命令帮助

在聊天框中打印所有可用命令的简要说明列表。

### 用法

```
/wb-help
```

### 无参数

此命令不接受任何参数。

### 返回内容（节选）

```
查询类:
  /wb-list                              — 列出所有世界书
  /wb-info <名称>                       — 查看世界书统计
  /wb-search name=<名> q=<词>           — 搜索条目

管理类:
  /wb-new <名称>                        — 创建世界书
  /wb-new-entry name=<名> [title=<标>]  — 创建条目
  /wb-del-entry name=<名> uid=<UID>     — 删除条目

批量字段操作:
  /wb-set-strategy name=<名> strategy=<策略> [uids=...]
  /wb-set-position name=<名> pos=<位置> [uids=...]
  ...

行为与工具:
  /wb-set-recursion name=<名> [pi=] [po=] [uids=...]
  /wb-set-effect name=<名> [sticky=] [cooldown=] [delay=] [uids=...]
  /wb-copy from=<名> to=<名> [uids=...]
  /wb-ui                                — 打开双面板管理界面
  /wb-help                              — 显示此帮助
```

## 递归与效果的关系图

```
关键字/向量命中条目 A
  │
  ├─ [Delay > 0]  → 等待 N 轮后再插入
  │
  └─ 插入上下文
       │
       ├─ [Sticky > 0]  → 接下来 N 轮内即使无命中也保持插入
       │
       ├─ [Cooldown > 0] → 插入后 N 轮内不再被激活
       │
       └─ 递归扫描（条目 A 的内容触发条目 B？）
              │
              ├─ [A.po=true]  → 不扫描（不触发递归）
              └─ [B.pi=true]  → B 不被递归触发
```
